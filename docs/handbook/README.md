# Handbook Overview

The handbook captures durable reference material for the Industry Resilience platform. Use these guides when you need process, operational, or architectural context:

- **Architecture** – `ARCHITECTURE.md` and `ARCHITECTURE_OVERVIEW.md` describe the layered design, data flow, and component responsibilities.
- **Operations** – `AUTOMATION.md`, `AUTOMATION_ROLES.md`, `STATUS.md`, and `SUPPORT.md` cover runbooks, historical status entries, and support expectations.
- **Product & Planning** – `PLAN.md`, `REPORT.md`, and `RELEASE_NOTES.md` preserve roadmap commitments and historical context.
- **Governance** – `SECURITY.md` documents vulnerability reporting and dependency-monitoring expectations.
- **Historical stewardship snapshot** – `STEWARDS_REPORT.md` contains dated measurements and recommendations; do not treat it as current CI or coverage evidence.

For current validation, use the latest GitHub Actions runs and the completed public-beta record in [issue #72](https://github.com/Nobodyworld/app-industry-resilience/issues/72). Tactical work history remains under `docs/execplans/`, and high-level stakeholder summaries remain under `docs/exec/`.
