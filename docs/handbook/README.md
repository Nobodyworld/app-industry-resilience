# Handbook Overview

The handbook captures the authoritative reference material for the Idiot Index platform. Use these guides when you need durable process or architectural knowledge:

- **Architecture** – `ARCHITECTURE.md` and `ARCHITECTURE_OVERVIEW.md` describe the layered design, data flow, and component responsibilities.
- **Operations** – `AUTOMATION.md`, `AUTOMATION_ROLES.md`, `STATUS.md`, and `SUPPORT.md` cover day-to-day runbooks and responsibilities.
- **Product & Planning** – `PLAN.md`, `REPORT.md`, and `RELEASE_NOTES.md` track roadmap commitments and historical context.
- **Governance** – `SECURITY.md` and `STEWARDS_REPORT.md` explain compliance requirements, audit expectations, and current health metrics.

If you are looking for tactical how-to steps for recent initiatives, check `docs/execplans/`. For high-level stakeholder summaries, see `docs/exec/`.
